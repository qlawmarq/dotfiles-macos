# Claudeコマンド: プルリクエストレビュー

このコマンドは、徹底的なコードレビューを実行し、プルリクエストに対して建設的なフィードバックを提供することを支援します。

## 使用方法

プルリクエストをレビューするには:

```
/pr-review 123
```

またはURLを使用:

```
/pr-review https://github.com/owner/repo/pull/123
```

引数が提供されない場合、ClaudeがレビューしたいPRを尋ねます。

## このコマンドが行うこと

1. `gh pr view <number>` を使用してPRの詳細を取得:
   - タイトルと説明
   - 変更されたファイル
   - 含まれるコミット
   - 現在のステータスとチェック
2. `gh pr diff <number>` を使用してdiffをレビュー
3. 変更を以下の観点から分析:
   - コード品質とベストプラクティス
   - 潜在的なバグや問題
   - パフォーマンスの考慮事項
   - セキュリティの懸念
   - テストカバレッジ
   - ドキュメントの完全性
4. PRがプロジェクトの規約に従っていることを確認:
   - コーディングスタイル
   - コミットメッセージ形式
   - PR説明の完全性
5. 構造化されたフィードバックを準備:
   - 全体的な評価
   - コード変更に対する具体的なコメント
   - 改善の提案
   - 必要に応じて明確化のための質問
6. オプションで `gh pr review <number>` を使用してレビューを投稿

## レビューフォーカスエリア

### コード品質
- 可読性と保守性
- プロジェクトコーディング基準への準拠
- 適切なエラーハンドリング
- 適切な抽象化

### 機能性
- ロジックの正確性
- エッジケースの処理
- 後方互換性
- 破壊的変更の文書化

### テスト
- テストカバレッジの適切性
- テストケースの包括性
- エッジケースのテスト

### ドキュメント
- コードコメントの明確性
- 必要に応じたREADMEの更新
- API変更の文書化

### セキュリティ
- ハードコードされたシークレットがないこと
- 入力検証の存在
- SQLインジェクション防止
- XSS防止

## レビューガイドライン

- **建設的に**: 作成者を批判するのではなく、コードの改善に焦点を当てる
- **具体的に**: 正確な行を指摘し、懸念事項を説明する
- **解決策を提案**: 問題を指摘するだけでなく、代替案を提供する
- **質問する**: 何かが不明確な場合は、明確化を求める
- **良い仕事を認める**: よく書かれたコードや巧妙な解決策を強調する
- **区別する**: 修正必須の問題と改善提案を区別する
