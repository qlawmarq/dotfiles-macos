このプロジェクトは macOS のセットアップスクリプトです。
詳細は @README.md や @CLAUDE.md に記載されています。

## タスク概要

- Claude Code モジュールへの PRPs-agentic テンプレートの導入

## 背景・目的

- このプロジェクトは macOS 向けの初期セットアップスクリプトで、Claude Code を利用するための設定やスクリプトも含まれています。 @modules/claude/apply.sh
- 既にいくつかのコマンドやフックのテンプレートがあります。ここに PRP（Product Requirement Prompt）フレームワークを有効活用するためのテンプレートを追加してもらいます。
- PRP（Product Requirement Prompt）フレームワークはプロンプトエンジニアリングの手法の一つで、設計書（要件定義）をもとに LLM にコードを生成してもらうためのフレームワークです。`./PRPs-agentic-eng`に他のプロジェクトのサンプルがあります。
- まず、PRP フレームワークの使い方を把握してください。その上で、Claude Code を用いて汎用的に PRP フレームワークをあらゆるプロジェクトで活用するための方法を検討してください。
- Claude Code はユーザー単位の設定（`~/.claude`）とプロジェクト単位の設定（`XX_PROJECT/.claude`）を使い分けることができます。ユーザー単位の設定を使うことが望ましい場合は`~/.claude`をそのまま活用し、プロジェクト単位の設定が望ましい場合はユーザー単位の設定からプロジェクト単位の設定を LLM に生成してもらうことになります。
- プロジェクト単位の設定が必要な場合、 @modules/claude/commands/hooks/setup-hooks.md を参考にしてください。ユーザー単位の設定で十分な場合は @modules/claude/commands/git/commit.md を参考にしてください。
- LLM 活用のベストプラクティスに従ったプロンプトを用意してもらう必要があります。

## 作業ルール

**以下のルールは絶対に遵守してください:**

### 1. 憶測での作業の禁止

- **絶対に憶測で作業しないこと**
- 信頼できる情報源のみを使用すること:
  - ✅ 公式ドキュメント
  - ✅ 実際のコード
  - ✅ 実行ログ・エラーメッセージ
  - ✅ テスト結果
  - ❌ 推測や想像
  - ❌ 未確認の仮説

(`context7`や`fetch` MCP 関数、あるいは Web 検索を活用することで情報を収集してください。情報にアクセスできない場合はユーザーに情報のコピーを要求してください。)

### 2. 不明点の質問義務

- **分からないことがある場合、実装を開始する前に必ず質問か調査をすること**
- 質問すべき状況:
  - 仕様が不明確な場合
  - エラーの原因が特定できない場合
  - 複数の実装方法があり、どれが適切か判断できない場合
  - 既存コードの意図が理解できない場合
  - ドキュメントとコードに矛盾がある場合
- 調査すべき内容:
  - ライブラリの使い方が不明な場合
  - 実際の出力・入力インターフェースが不明な場合

### 3. 実装前の確認

- コードを書く前に、以下を確認すること:
  - 該当箇所のコードを実際に読んで理解している
  - 既存の実装パターンを調査している
  - ライブラリなどの実際のインターフェース情報を把握していること
  - 影響範囲を特定している

### 4. ドキュメント参照

- 作業前に以下のドキュメントを確認すること:
  - `CLAUDE.md`: プロジェクト全体のアーキテクチャと開発ガイド
  - `AGENTS.md`: リポジトリガイドラインと開発ワークフロー
  - `README.md`: ユーザー向けドキュメント

## 手順

- まず、既存のコードなどプロジェクトについて具体的に理解をしてください。
- 関連するライブラリについての公式情報も詳細に収集してください。
- その上で、要件を満たすためのプランを考えて、提案してください。プランは複数提案することもできます。
- 憶測で提案や作業を行わないように情報の調査を入念に行ってください。
- 提案したプランをユーザーが承認することで実装などの実際の作業に入ることができます。
