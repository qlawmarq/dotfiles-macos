# Claudeコマンド: プルリクエスト作成

このコマンドは、包括的な説明を含む構造化されたプルリクエストの作成を支援します。

## 使用方法

プルリクエストを作成するには、以下を入力してください:

```
/pr-create
```

## このコマンドが行うこと

1. 現在のブランチを確認し、main/masterブランチでないことを確認
2. 変更がコミットされていることを確認（必要であれば先に `/commit` を使用）
3. `git log` と `git diff` を実行して、PRに含まれる全てのコミットを理解
4. 変更を分析して包括的なPR説明を作成:
   - 変更の目的を要約
   - 主要な変更と改善点をリスト化
   - 破壊的変更を特定
   - 依存関係や関連イシューを記載
5. プルリクエストテンプレート（`.github/pull_request_template.md`）の存在を確認し、その形式に従う
6. まだプッシュされていない場合、現在のブランチをリモートにプッシュ
7. `gh pr create` を使用してPRを作成:
   - 明確で説明的なタイトル
   - テンプレートまたはベストプラクティスに従った包括的な説明
   - 該当する場合は適切なラベル
8. レビュー用のPR URLを表示

## プルリクエストのベストプラクティス

- **明確なタイトル**: 変更を1行で要約（例: "OAuth2によるユーザー認証を追加"）
- **包括的な説明**: 以下を含める:
  - どのような変更が行われたか
  - なぜ変更が行われたか
  - 変更をテストする方法
  - 破壊的変更またはマイグレーション注意事項
  - 関連イシューまたは依存関係
- **テンプレートに従う**: プロジェクトのPRテンプレートがあれば使用
- **イシューをリンク**: `#issue-number` を使用して関連イシューを参照
- **レビューを依頼**: 必要に応じて適切なレビュアーをタグ付け

## 要件

- `gh` CLIがインストールされ、認証されていること
- 現在のブランチにプッシュするコミットがあること
- PR作成前に変更がコミットされていること
